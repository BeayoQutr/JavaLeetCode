# LeetCode 180 天系统刷题路线（Java）

> 目标：系统掌握常见数据结构与算法，巩固 Java 解题基础，并建立可复用的分析、编码和复盘能力。

## 使用说明

- 全程共 180 天，每天 2 题，共 360 次训练任务；复习日会重做代表题，实际覆盖约 300 道不同题目。
- 这里的 180 天指 **180 个编号训练日**，不要求对应连续的 180 个自然日。缺勤后从下一个 Day 继续，禁止为了追进度一天补做多个 Day。
- 默认按题目当前主站难度标注：简单、中等、困难。难度只表示题目综合门槛，不代表每天必须一次通过。
- 普通题独立思考 25～35 分钟，困难题独立思考 40～50 分钟。超时后先看提示或关键思路，再独立完成代码。
- 每题至少记录：核心思路、时间复杂度、空间复杂度、易错点、是否需要复习。
- 复习题不要复制旧代码，应先口述思路，再从空文件重新实现。
- 固定复习日的两题是默认题单，刻意混合近期与跨周内容，并不只复习上一周；若当天有到期错题，用最高频错题替换第二道默认复习题。
- 错题的次日复习只做 10～15 分钟的口述、手算和关键代码回忆；第 7 天、第 30 天再完整重写，优先放入固定复习日，避免无限叠加题量。
- 通常每天预计 60～90 分钟；含困难题约 90～120 分钟，双困难日可把同一个 Day 拆到两个自然日完成，但不要跳题。
- 推荐使用 Java 17 或项目已有 Java 版本；优先掌握数组、`StringBuilder`、`HashMap`、`HashSet`、`ArrayDeque`、`PriorityQueue` 和比较器。

## 每日完成标准

1. 阅读题意并手算至少一个样例。
2. 写出暴力思路，判断瓶颈，再推导优化方案。
3. 独立编码并通过测试。
4. 写明复杂度，补充空输入、单元素、重复元素、极值等边界用例。
5. 当天错题加入复习清单，按次日、第 7 天、第 30 天的间隔复习；若某阶段积压超过 6 题，暂停新 Day，先安排补弱日。

## 阶段验收标准

完成每个阶段后，不看旧代码完成以下验收：

- 能说出该阶段每种核心模型的适用条件、状态定义或不变量。
- 能为简单题在 15 分钟内、中等题在 35 分钟内写出可运行解法。
- 能解释解法为什么正确，而不是只复述代码流程。
- 能给出时间复杂度和空间复杂度，并说明主要开销来自哪里。
- 阶段代表题指该阶段所有标有 `[复习]` 的不同题目；在对应限时内独立二刷通过率达到 80% 才进入下一阶段。
- 未达到 80% 时插入 2～3 个补弱自然日，不增加 Day 编号；完成后再继续下一 Day，因此实际日历周期可以超过 180 天。

## Java 解题检查清单

- 整数求和、乘积、距离和下标差可能溢出时使用 `long`。
- 栈和队列优先使用 `ArrayDeque`，不要使用旧的 `Stack`。
- 堆使用 `PriorityQueue`；比较器优先用 `Integer.compare(a, b)`，避免直接相减溢出。
- DFS、回溯与树递归必须明确终止条件，并在回溯返回前恢复现场。
- 图题先确认是有向图还是无向图，再决定邻接表、入度数组和访问状态。
- 二分查找先定义搜索区间是闭区间还是左闭右开，并保持循环不变量一致。
- 动态规划先写清 `dp` 含义、初始状态、转移方程、遍历顺序和最终答案位置。
- 提交前测试空集合、单元素、全相同、严格递增/递减、最大输入和重复值。

## 卡题处理规则

1. 简单题 25 分钟、中等题 35 分钟、困难题 50 分钟仍无有效进展时停止硬耗。
2. 只看一级提示：使用的数据结构或算法模型，然后继续独立思考 10 分钟。
3. 仍无法完成时阅读思路，不直接复制代码；关闭题解后从头实现。
4. 当天标记为错题，并安排在次日、第 7 天和第 30 天重做。
5. 同一模型连续错两题时，暂停新题，先总结模板和易错点。
6. 阅读思路后仍无法独立实现时，先手算样例并写伪代码，次日从空文件再做；若完整复习再次失败，则退回该模型两道更基础的题，连续独立通过后恢复新题。
